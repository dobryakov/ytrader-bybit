# Проект - торговая система на микросервисной архитектуре

## Цель

Создать масштабируемую, отказоустойчивую, модульную торговую платформу для автоматической торговли криптовалютами на бирже Bybit с постепенным развитием.

## Общая архитектура

Проект состоит из набора микросервисов в контейнерах docker (под общим docker-compose.yml), каждый из которых отвечает за отдельный функциональный блок, например:

- **Микросервис агрегации данных WebSocket** `ws-service.md` — централизует подключение к WebSocket Bybit, управляет подписками, фильтрует данные и рассылает события через очереди.
- **Микросервис управления ордерами (Order Manager)** — выставляет, отслеживает ордера, управляет позицией и балансом, хранит данные в общей базе.  
- **Микросервис модели (Model Training Service)** — обучает и дообучает торговую модель на данных из общей базы и событий, передаёт обновления модели ордер-менеджеру.  
- **Другие сервисы** — мониторинг, UI, интеграции и прочие (будут разрабатываться поэтапно).

## База данных

- Все микросервисы используют одну общую базу данных PostgreSQL для обмена и хранения данных.
- Рекомендуется использовать специализированные базы данных под конкретные задачи (например, векторный поиск или обучение ML моделей).

## Взаимодействие микросервисов

- События и команды между сервисами передаются через очереди с разделением по сущностям и классам (предпочтительный способ).
- В особых случаях, для запроса данных, или для удобства разработки - допускается реализовывать HTTP REST API.

## Общие требования

- используй актуальную документацию и SDK Bybit, а также mcp context7 для принятия правильных решений.
